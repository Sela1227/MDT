# MDT 會議管理系統 使用說明書

> 給彰濱秀傳癌症中心三位個管師的快速上手指南。
> 版本對應:**V4.6.3**(2026/04)

---

## 一、登入與切換身份

### 第一次進入系統

打開系統網址後會看到登入畫面,選擇您的姓名:

- 楊靜雯(分機 17965) — 血液淋巴癌、消化道癌、婦癌
- 郭美伶(分機 17379) — 胸腔癌、乳癌
- 林伯儒 — 頭頸癌、肝膽胰癌、泌尿道癌

選定身份後,系統會記住「哪些會議是您建立的」 — 這跟刪除權限有關(只能刪自己建立的會議)。

### 切換身份

左側選單最上方點您的姓名 → 切換身份。

## 二、開會前的準備

### 1. 建立新會議

主畫面點「**建立新會議**」 → 選擇會議日期、時間、地點 → 勾選參與的癌別。

> **典型情境**:每週的乳癌+胸腔癌聯合 MDT,郭美伶建立會議時兩個癌別都勾起來,系統會自動產出兩個癌別的個案分頁。

### 2. 填個案資料

每個癌別會有一個分頁,內含三大區塊:

| 區塊 | 用途 |
|------|------|
| **前期追蹤** | 上次會議討論過、本次要回報進度的個案 |
| **個案討論** | 本次新提出的個案 |
| **特殊議題** | 病理討論、影像討論等不依個案的主題 |

**填寫小技巧**:
- 病歷號可直接從病歷系統複製貼上
- 醫師欄位是下拉式,自動從主檔讀取(科別 → 姓名)
- 病理切片可在「附加影像」內上傳,系統會自動分類

### 3. 自動帶入上次追蹤

系統會自動把上次會議標記為「繼續追蹤」的個案帶到本次的「前期追蹤」區。  
**如果您不想要某個案被自動帶入**,在上次會議的會後填寫時把該個案的「下次追蹤」設為「結案」即可。

---

## 三、會議當下:產出資料

填完個案後,進到**右側「產出」區**,從上而下依序是:

### 產出區結構(V4.6.2 之後)

```
[資料夾授權狀態][病理切片影像集中]
[會後填寫]                              ← 開會時也能用,記錄當場決議
投影片配色 [●●][●●][●●][●●][●●]      ← V4.6.1 加,V4.6.2 上移
[LINE通知][HTML投影片][HTML分享][PPTX簡報][DOCX記錄]
[Excel匯出/匯入][JSON匯出/匯入]
[AI 提示詞][AI 生成摘要]
```

### 1. LINE 通知

複製到剪貼簿,直接貼到 MDT 群組通知參與醫師。

### 2. HTML 投影片(會議當天投影用)

按下後在新分頁開啟全螢幕投影片,鍵盤左右鍵翻頁、F 鍵全螢幕、放大鏡功能可細看影像。

> **V4.6.0 起新功能 — 投影片配色**  
> 在「HTML 投影片」按鈕**正上方**有一條色票橫條,5 個風格可選:  
> **彰濱經典**(深灰藍,預設) / **暖陽**(暖棕橘) / **森林**(沉穩綠) / **薰衣草**(柔和紫) / **高對比**(黑灰白,適合列印)  
> 點一下色票即時換色,系統會記住您的選擇,**三位個管師可以各選不同偏好**互不影響。

### 3. HTML 分享

把投影片上傳到 GitHub 產生分享連結,給醫師遠端觀看(需在「設定 → 系統」先設定 GitHub Token)。

### 4. PPTX 簡報

下載可在 PowerPoint 編輯的版本。

### 5. DOCX 會議記錄

下載「會議決議確認單」格式的 Word 檔。

## 四、會後填寫(V4.3.43 加)

會議結束後,點產出區的「**會後填寫**」按鈕(藍色大按鈕),會打開全螢幕填寫面板。

每個個案需要填:
- **摘要**:本次討論重點
- **結論**:會議決議
- **下次追蹤**:結案 / 繼續追蹤  ← **影響下次會議是否自動帶入**

填完按儲存,系統會把這些資料寫回會議記錄。

## 五、儲存、刪除、複製會議

### 儲存

每次點「儲存會議」會寫入瀏覽器、並推送到 NAS(若有設定)。每次成功儲存系統會跳「已儲存」訊息。

### 刪除

兩個刪除入口:
- **列表刪除**:左側「最近會議」每筆右側的垃圾桶圖示
- **會議內刪除**:閱覽模式右上角的「刪除此會議」按鈕

> **權限控制(V4.3.40 起)**  
> 只能刪除「自己建立的」會議。試圖刪別人建立的會議會跳出提示,告訴您原建立者是誰。

> **批次刪除(V4.3.43 起)**  
> 在歷史紀錄頁可勾選多筆批次刪除。**只要勾到任何一筆不是您建立的會議,整批會被擋下**並列出哪幾筆 — 取消勾選那幾筆才能繼續。

### 複製會議(V4.3.43 加)

閱覽模式右上角的「複製為新會議」 — 用舊會議當模板開新的(個案、醫師資料保留,日期清空)。

## 六、跨機器同步(V4.3.41-V4.4.0)

如果您在診間用 A 電腦填、回辦公室用 B 電腦看,系統會透過 NAS 自動同步:

- **登入後自動同步**:從 NAS 拉最新版本到本機
- **存檔後自動推送**:把變動推到 NAS
- **跨機刪除傳播**(V4.4.0):A 電腦刪除的會議,B 電腦下次同步時會跟著刪除
- **保留期 90 天**:被刪除的會議在 NAS 保留 90 天才完全清除(避免誤刪無法救)

> **第一次設定**:設定 → 備份 → 選擇 NAS 資料夾,授權後即可。

## 七、設定維護

### 1. 醫師管理(設定 → 醫師)

- 依科別分組,可展開/收合
- 每個科別標頭列有 4 顆按鈕:**▲ ▼**(科別整組上移/下移,V4.5.0 加)、**重命名**、**刪除科別**
- 每位醫師也可以**▲ ▼ 上下排序**、編輯、刪除
- 加新醫師:選科別 + 填姓名(系統自動加「醫師」兩字)

### 2. 重新套用主檔遷移(V4.4.0 加)

如果系統升級後發現「科別命名沒更新」(例如還是舊的「頭頸外科」、「消化內科」),  
進「設定 → 醫師」 → 頂端藍色提示框 → 點「**重新套用主檔遷移**」按鈕。

系統會把舊版命名改新、補入新加的醫師,跳出統計訊息(重命名 N 位、新增 M 位)。**手動新增的醫師不會被刪除**。

### 3. 地點(設定 → 地點)

管理會議室清單。

### 4. 癌別(設定 → 癌別)

每個癌別可調整:
- 顯示名稱、配色
- 預設召集人、預設地點
- 預設參加成員
- **預設討論主題**(影像討論、病理討論等)

### 5. AI 摘要(設定 → 系統 → AI 摘要設定)

填入 OpenAI 或 Claude API Key 後,可在產出區用「AI 生成摘要」自動產出個案摘要。

### 6. GitHub 投影片分享(設定 → 系統 → GitHub)

填入 GitHub Personal Access Token 後可使用「HTML 分享」功能。

## 八、常見問題

### Q1. 同事改的我看不到?

如果您跟同事都用 NAS 同步:存檔後 1-2 分鐘內推上去,對方下次登入會拉下來。  
**手動觸發**:設定 → 備份 → 立即同步。

### Q2. 不小心關了瀏覽器,資料會不見嗎?

不會。資料存在瀏覽器(localStorage),關掉重開仍在。但**清快取會清掉本機資料** — 養成習慣定期看 NAS 是否有備份。

### Q3. 為什麼某個個案沒被自動帶入下次?

兩種可能:
1. 上次會議的「會後填寫」把它的下次追蹤設為「**結案**」 — 設計如此
2. 該個案不在「前期追蹤」區裡(只有那一區才會追蹤)

### Q4. 投影片字太小看不清?

HTML 投影片內有放大鏡功能(R 鍵或點擊圖片);若整體字小,可用瀏覽器縮放(Ctrl + 滾輪)。

### Q5. 想印投影片?

選**高對比**配色(黑灰白)再產出 → 用瀏覽器列印或產 PPTX 後在 PowerPoint 列印。

### Q6. 想復原一個誤刪的會議?

如果您 90 天內發現誤刪,且 NAS 上還有 tombstone 標記未清除 — 聯絡資訊人員手動處理。**從前端介面無法直接復原**,所以刪除前請務必確認。
